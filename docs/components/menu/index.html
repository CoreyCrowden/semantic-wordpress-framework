Alterations to WordPress

.menu.item

To maintain semantic coding the following code must be added to the themes functions.php file to allow native nav menus to use the menu classes.

And be sure to declare the .menu class when outputting the menu.

Initiating a menu

.menu.item

Creating a menu is simply a matter of calling the .menu class. Of course WordPress does this by default so you won't even have to worry about that unless it's a manual menu.

It's important to remember that the menu class uses the Flex Grid System and all of it's classes. By sharing classes this makes the menu system light weight and incredibly powerful. Please refer to the Flex Grid System for class references.

The default menu is a simply arrangement of links in a horizontal layout.

Creating complex menus through SASS

sass

While having all these menu options is great, having to manually enter classes at a content level is not usually preferably. But luckily through the use of the sass @extend function we can define the menu styles simply and without code repetition.

Working in our project scss file we need to target specific menu locations and then specify the menu level to apply the @extend. The example below shows how to build a horizontal menu with a dropdown at the first sub-menu level followed by a slide out at the second sub-menu level and finally formatted text at the third sub-menu level.

Note: It may also be a good idea to @extend a scaling class like @extend .small to the sub-menu.
